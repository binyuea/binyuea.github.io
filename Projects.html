2-way R10K : An out of order true renaming register processor

In this project, I'm in charge of RS, I/D cache, fetch, prefetch, predictor, CDB.
  • Dcache: Using 2-way set associate cache and LRU(refer, dirty bit) algorithm to determine which block will be evicted to memory can decrease the usage for I/O of memory.
  • Icache: Directed map cache, easy to implement, and it fits to instruction fetch features.
  • Prefetch: For higher usage I/O of memory, prefetch allocate I/O accroding to this priority order(from high to low): Dcache load, Icache load, Icache prefetch, Dcache store.
  • RS(reserve station): Recording each instruction state, and determine which instruction to issue. I allocated different blocks to corresponding insn. In this way, we can issue at most 6 instructions(2 alu, 1 store, 1 load, 1 mult, 1 branch), which can improve system potential performance.
